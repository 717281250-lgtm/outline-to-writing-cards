# V0.1 测试清单

运行 Skill 后，请不要先看它“聪不聪明”，先检查它有没有让你更容易动笔。

## 一、结构检查

- [ ] 是否真的生成了 3 篇独立短稿，而不是背景 / 案例 / 结论三段？
- [ ] 每篇是否都能单独回答一个问题？
- [ ] 每篇是否都有一句明确判断？
- [ ] 每篇是否都有自己的收束，而不是只给下一篇铺垫？

## 二、减负检查

- [ ] 是否明确删除了一批原大纲内容？
- [ ] 是否把“可以写”压缩成了“必须写”？
- [ ] 整篇是否最多只留下 3 个重研究任务？
- [ ] 每篇核心证据是否不超过 3 项？
- [ ] 每篇是否只有 1 个深挖点？

## 三、作者所有权检查

- [ ] “作者真正感兴趣的是”是否准确？
- [ ] 是否有一张卡让你产生“这篇我现在就能写”的感觉？
- [ ] 是否仍然存在明显的 AI 正确废话？
- [ ] 是否有某张卡只是因为“重要”而存在，而不是因为值得写？

## 四、启动检查

- [ ] 是否明确推荐了先写哪一篇？
- [ ] 推荐理由是否符合你的兴趣和现有素材，而不是按顺序？
- [ ] “现在只做的第一步”是否能在 15—30 分钟内完成？
- [ ] 完成条件是否能阻止你继续无限补资料？

## 六、记录问题

1. 哪张卡最想写？为什么？
2. 哪张卡仍然像 AI 大纲？
3. 哪些被删除的内容你仍然舍不得？
4. 哪个字段对启动最有帮助？
5. 哪个字段显得多余？
6. Skill 是否低估或高估了某项研究负担？
7. 你实际开始写后，卡片在哪一步失效？
