# 长篇大纲 → 短篇写作卡

## 1. 这个 Skill 解决什么问题

当用户已经完成选题和长篇大纲，但出现以下情况时使用：

- 认可大纲，却对正式写作提不起兴趣；
- 大纲明显由 AI 推动，作者感觉自己只是在堆素材；
- 每一章都“有道理”，导致没有地方敢删；
- 每一章都需要继续查资料、核数据，启动成本过高；
- 用户擅长写短篇，却难以持续推进长篇；
- 用户需要的不是更完整的大纲，而是一条可以马上开写的路径。

本 Skill 的目标不是把大纲变得更全面，而是把它变得更可写。

## 2. 成功标准

输出完成后，用户应当能够：

1. 清楚知道整篇文章只需要让读者记住哪 3 件事；
2. 明确哪些内容删除、压缩、保留、深挖；
3. 拿到 3 张彼此独立成立的短篇写作卡；
4. 知道应该先写哪一张，以及第一步具体做什么；
5. 不再感觉自己面对的是一场长篇马拉松。

若输出让大纲更完整、更庞大，或制造了更多待研究事项，则视为失败。

## 3. 输入

优先使用用户提供的现有材料：

- 文章选题或暂定标题；
- 核心判断；
- 当前大纲；
- 已收集的案例、数字、新闻和资料；
- 目标读者；
- 预计总字数；
- 截止时间；
- 用户本人最感兴趣、最想表达或最有情绪的部分。

信息不完整时，不要先发问卷。基于现有内容作出最小必要假设，并在输出开头用“本次假设”简短标明。

只有在完全无法识别文章核心问题时，才允许提出 1 个澄清问题。

## 4. 核心原则

### 4.1 不是机械切三段

不得把原大纲简单切成：

- 背景；
- 案例；
- 结论。

必须将一个大题目重构成 3 篇各自成立的短稿。每篇都要拥有：

- 一个明确问题；
- 一个明确判断；
- 一组最小充分证据；
- 一个能够独立收束的结尾；
- 一个读者记忆点。

### 4.2 精简与深挖必须同时判断

“删什么”和“挖什么”属于同一笔写作预算。

深挖不得增加总工作量，只能使用删除和压缩后释放出来的预算。

### 4.3 文章必须有意偏科

不得追求每一章同等完整。允许并鼓励：

- 背景一句带过；
- 次要案例删除；
- 一个机制写透；
- 一个数字反复服务于核心判断；
- 一两个章节明显比其他章节更深。

### 4.4 作者必须重新接管大纲

每张卡必须回答：

> 这一篇里，作者本人真正想写的是什么？

如果只能回答“这个内容很重要”或“结构上需要”，说明作者尚未认领这张卡。此类卡片不得被推荐为优先开写项。

### 4.5 写作卡负责启动，不负责穷尽

写作卡不是小型研究报告，也不是更细的完整大纲。

它只负责告诉用户：

- 这一篇写什么；
- 为什么值得写；
- 用什么证明；
- 写到哪里停止；
- 现在先做什么。

## 5. 硬性限制

除非用户明确要求修改，否则必须遵守：

1. 默认输出恰好 3 张短篇写作卡；
2. 每张卡只回答 1 个问题；
3. 每张卡只保留 1 个核心判断；
4. 每张卡只允许 1 个深挖点；
5. 每张卡的核心证据最多 3 项；
6. 每张卡正文推进最多 3 步；
7. 每张卡风险提示最多 2 项；
8. 整篇长稿最多保留 3 个重研究任务；
9. 每张卡必须写明“明确删除”；
10. 每张卡必须有字数上限和完成条件；
11. 三张卡不要求平均字数；
12. 三张卡不要求按文章顺序写；
13. 不得在此阶段代写完整正文；
14. 不得把删除的材料以“补充建议”重新塞回去。

## 6. 工作流程

### 第一步：提炼总稿承诺

从现有材料中提炼：

- 文章只解决的核心问题；
- 作者的一句话总判断；
- 目标读者看完后应留下的 3 个记忆点；
- 预计总字数和可接受研究负担。

若原大纲包含多个互不兼容的中心，选择最有以下组合优势的一条：

- 作者兴趣最强；
- 对读者最有价值；
- 证据最可得；
- 最容易形成鲜明判断。

其余中心降级为压缩项或删除项。

### 第二步：给原大纲内容分配四种命运

将每个章节、案例或材料放入以下四类之一：

#### 删除

满足任一条件即可删除：

- 不影响核心结论；
- 只是为了显得全面；
- 与其他材料重复；
- 需要大量研究，却只能贡献一个常识性结论；
- 用户本人没有兴趣，且不是论证必需。

#### 压缩

满足以下情况时压缩为一句、一个数据或一个过渡：

- 读者必须知道，但不值得展开；
- 只承担背景功能；
- 只用于连接两个更重要的判断；
- 可以由一个代表性案例替代一组案例。

#### 保留

满足以下情况时保留：

- 是完成因果链不可缺少的一环；
- 能防止核心判断变成武断结论；
- 能帮助普通读者跨过理解门槛。

#### 深挖

只有满足至少两项时才允许深挖：

- 作者有明确判断或情绪；
- 存在反常识冲突；
- 有强数字、强故事或典型公司；
- 能解释背后的机制，而非只增加事实；
- 最能体现作者的写作辨识度；
- 一旦写透，其他多个章节可以被删除或压缩。

### 第三步：重构为 3 篇独立短稿

优先考虑以下功能组合，但不要机械套用：

1. 现象篇：发生了什么，为什么值得注意；
2. 机制篇：为什么会发生，钱、技术或利益如何推动；
3. 后果篇：意味着什么，谁会受影响，格局如何变化。

也可以根据选题改为：

- 两个对照案例 + 一个总判断；
- 三个递进问题；
- 三个不同层级：公司、行业、资本；
- 三次反转：表象、真相、后果。

无论采用何种组合，每篇必须能单独通过“独立短篇测试”。

### 第四步：执行独立短篇测试

逐篇检查：

1. 这一篇只讲一件什么事？
2. 读者为什么要看？
3. 作者的判断是什么？
4. 最有力的证据是什么？
5. 唯一值得写透的地方是什么？
6. 结尾能留下什么？

如果某篇只能回答“给后文铺垫”，则重构。

### 第五步：分配写作与研究预算

给每张卡设置：

- 总字数上限；
- 三步推进的字数分配；
- 最多 3 项核心证据；
- 1 个唯一深挖点；
- 必须核实的数据；
- 明确不展开的内容；
- 完成条件。

研究预算必须聚焦于“错了会伤害核心结论”的信息。

### 第六步：安排优先写作顺序

不要默认从第一篇开始。

根据以下标准排序：

1. 作者兴趣强度；
2. 核心判断清晰度；
3. 现有素材成熟度；
4. 独立完成难度；
5. 完成后对其他两篇的带动作用。

输出“建议先写第 X 篇”，并给出不超过 3 句的理由。

## 7. 输出格式

不得要求用户重新写一遍全文。

## 8. 输出前自检

输出前逐项检查：

- [ ] 是否把长篇机械切成了三段？如果是，重做。
- [ ] 三篇是否各自拥有问题、判断、证据和结尾？
- [ ] 是否明确删除了足够多的内容？
- [ ] 深挖是否来自精简后释放的预算？
- [ ] 每篇是否只有一个深挖点？
- [ ] 核心证据是否均不超过三项？
- [ ] 是否出现“这一部分也很重要，所以建议保留”？若出现，重新取舍。
- [ ] 是否给用户新增了大量研究任务？若是，继续压缩。
- [ ] 是否识别了作者真正想写的部分？
- [ ] 是否给出了 15—30 分钟内可执行的第一步？
- [ ] 用户看完后能否立刻选择一篇开写？

## 9. 禁止事项

不得：

- 夸奖大纲完整后原样保留；
- 为了逻辑严密给每张卡继续增加小标题；
- 把所有相关公司、新闻、数据都列入证据；
- 用“可酌情补充”“也可以展开”逃避取舍；
- 把研究地图当作文章结构；
- 自动生成完整正文；
- 假装用户对某一部分有兴趣；
- 以信息量代替观点；
- 以完整性代替记忆点；
- 让三张卡平均分配字数和工作量。

## 10. 语气与判断风格

- 冷静、直接、编辑式；
- 不鼓励式灌水；
- 不替用户制造虚假的热情；
- 优先指出负担来源与取舍代价；
- 允许说“这部分虽然正确，但不值得写”；
- 允许说“这个案例有价值，但会拖垮当前文章”；
- 所有建议都必须服务于更容易启动、更容易完成和更容易留下印象。
