# outline-to-writing-cards

`outline-to-writing-cards` 是一个 Codex skill，用来把一份已经确定、但过重或难以启动的长篇大纲，压缩并重构为 3 张可以独立完成的短篇写作卡。

它不追求让大纲更完整，而是让文章更可写。

## 适用场景

当你已经有选题和长篇大纲，但出现这些情况时使用：

- 认可大纲，却对正式写作提不起兴趣
- 大纲明显由 AI 推动，作者感觉自己只是在堆素材
- 每一章都“有道理”，导致没有地方敢删
- 每一章都需要继续查资料、核数据，启动成本过高
- 你擅长写短篇，却难以持续推进长篇
- 你需要的不是更完整的大纲，而是一条可以马上开写的路径

## 输出目标

成功输出后，你应该能拿到：

- 这篇文章真正只需要让读者记住的 3 件事
- 哪些内容删除、压缩、保留、深挖
- 3 张彼此独立成立的短篇写作卡
- 最应该先写哪一张
- 现在 30 分钟内只做什么

如果输出让大纲更庞大、制造更多待研究事项，就算失败。

## 核心原则

### 不是机械切三段

不把原大纲简单切成“背景、案例、结论”。每张卡都要是一篇能独立成立的短稿，拥有自己的问题、判断、证据和结尾。

### 精简和深挖必须同时判断

“删什么”和“挖什么”是同一笔写作预算。深挖不能增加总工作量，只能使用删除和压缩后释放出来的预算。

### 文章必须有意偏科

不追求每一章同等完整。允许背景一句带过、次要案例删除、一个机制写透、一个数字反复服务于核心判断。

### 作者必须重新接管大纲

每张卡都必须回答：这一篇里，作者本人真正想写的是什么？

## 安装

把整个目录放到 Codex skills 目录：

```bash
cp -R outline-to-writing-cards ~/.codex/skills/
```

安装后，在 Codex 里使用：

```text
$outline-to-writing-cards
```

也可以直接贴一份长篇大纲，并要求“把它拆成 3 张写作卡”。

## 建议测试指令

```text
使用 outline-to-writing-cards Skill 处理下面的长篇计划。严格执行删减，不要代写正文，不要因为材料很多就全部保留。输出后告诉我最应该先写哪张卡，以及现在 30 分钟内只做什么。
```

## 暂不解决

- 自动联网搜集和核实新闻数据
- 深挖章节的完整研究工作
- 三篇完成后的全文润色
- 自动代写整篇长文

这些功能应该在本 skill 验证有效后，再拆成后续 skill。
